# Monster-Effective Tool Instructions

Based on cognitive psychology research, this enhanced description uses proven techniques for maximum effectiveness:

## WHAT (Core Purpose)
**Sequential Thinking**: A dynamic, reflective problem-solving tool that adapts as you think.

## THE POWER PATTERN (When to Use)
Use this tool when you need to **SOLVE**, **PLAN**, **ANALYZE**, or **DESIGN**:

### ✓ SOLVE: Complex multi-step problems
### ✓ PLAN: Projects with unknown scope  
### ✓ ANALYZE: Issues requiring course correction
### ✓ DESIGN: Solutions that need revision

## THE SUPERPOWERS (Key Features)

### 🔀 FLEX: Adjust, revise, branch as needed
- Change total_thoughts anytime
- Question previous thoughts
- Add thoughts beyond original estimate
- Express uncertainty freely

### 🧬 EVOLVE: Non-linear thinking paths
- Branch into alternatives
- Backtrack when needed  
- Revise past insights

### 🔬 VERIFY: Track and test ideas
- Generate hypotheses
- Verify systematically  
- Track results clearly

## PARAMETER QUICK REFERENCE (The Toolkit)

### Essential Parameters
- `thought` → Your current thinking step (text)
- `next_thought_needed` → Continue? (true/false)
- `thought_number` → Current position (number)
- `total_thoughts` → Estimated total (flexible)

### Branch Control
- `is_revision` → Revising previous? (true/false)
- `revises_thought` → Which thought# to revise
- `branch_from_thought` → Branch start point  
- `branch_id` → Branch identifier
- `needs_more_thoughts` → Need to extend? (true/false)

### Hypothesis Tracking
- `thought_type` → 'hypothesis' or 'verification'
- `related_to` → Which thought#s this relates to
- `verification_result` → 'confirmed'/'refuted'/'partial'/'pending'

## THE WORKFLOW (Your Action Plan)

### START → EXPLORE → VERIFY → CONCLUDE

1. **START**: Estimate thoughts needed (flexible!)
2. **EXPLORE**: Think, revise, branch freely  
3. **VERIFY**: Test hypotheses systematically
4. **CONCLUDE**: Complete with full summary

## THE RULES (Remember These)

1. **ESTIMATE FIRST**: Start with a thought count (adjust later)
2. **REVISE FREELY**: Question anything, anytime
3. **EXTEND ALWAYS**: Add thoughts beyond estimates
4. **MARK CLEARLY**: Label revisions and branches
5. **IGNORE NOISE**: Skip irrelevant information
6. **HYPOTHESIS EARLY**: Identify solutions quickly
7. **VERIFY THOROUGHLY**: Link verifications to hypotheses
8. **ITERATE UNTIL**: Continue until satisfied
9. **SUMMARIZE COMPLETELY**: Show all hypothesis states
10. **END DELIBERATELY**: Only stop when truly done

## MEMORY AIDS (Mnemonics)

**REVI-CHECK** for Key Actions:
- **R**evise previous thoughts
- **E**xtend when needed
- **V**erify hypotheses  
- **I**dentify branches
- **C**onnect related thoughts
- **H**alt only when complete
- **E**valuate thoroughly
- **C**onclude with summary
- **K**eep context always

## COGNITIVE LOAD OPTIMIZATION

### One Focus at a Time
- Process ONE thought per call
- Keep thoughts concise (3-7 concepts max)
- Use clear, action-oriented language

### Visual Structure
- Bullet points for multiple ideas
- Numbers for sequences
- Headers for major shifts

### Pattern Recognition
- Similar thought types get similar treatment
- Consistent parameter usage
- Predictable workflow steps

## ERROR PREVENTION

### Common Pitfalls to Avoid
1. ❌ Ending too early
2. ❌ Not labeling revisions
3. ❌ Skipping hypothesis marking
4. ❌ Breaking connection chains
5. ❌ Forgetting to link verifications

### Success Indicators
1. ✅ Each thought builds meaningfully
2. ✅ Revisions are clearly marked
3. ✅ Hypotheses are identified  
4. ✅ Verifications reference hypotheses
5. ✅ Summary shows complete status

## PERFORMANCE TIPS

### Chunking Strategy
- Group related thoughts in branches
- Limit 3-5 thoughts per reasoning chain
- Break complex ideas into steps

### Spaced Processing  
- Allow thoughts to evolve naturally
- Don't force immediate completion
- Use revision cycles for depth

### Active Monitoring
- Track hypothesis states actively
- Verify early and often
- Summarize major milestones

## EXAMPLES IN ACTION

### Basic Thought Flow
```
1. Initial analysis (thought_type=null)
2. Hypothesis identified (thought_type='hypothesis')
3. Testing approach (thought_type=null)
4. Verification result (thought_type='verification', related_to=[2])
5. Conclusion (next_thought_needed=false)
```

### Revision Pattern
```
1. Initial idea
2. Deeper analysis
3. REVISION: Better approach (is_revision=true, revises_thought=1)
4. Continue with improved direction
```

### Branching Structure
```
1. Main analysis
2. Option A exploration
3. Branch B start (branch_from_thought=1, branch_id='alt_solution')
4. Compare both approaches
5. Select best path
```

## FINAL DIRECTIVE

**Remember**: This tool adapts to YOUR thinking, not the other way around. Stay flexible, think clearly, verify thoroughly, and only stop when you're genuinely satisfied with the solution.
